# Accessibility, Privacy, and Safety Acceptance

- Every supported workflow is operable with keyboard and assistive technology without drag, hover, color discrimination, or time-critical precision.
- Result and consent changes use accessible semantics and restrained live announcements. Color, animation, position, icons, and biometrics are never the only security signal.
- Long technical values can be copied and inspected character by character without dominating the primary screen-reader flow.
- Sensitive screens restrict screenshots, recording, notifications, and app switching where supported. Clipboard use is explicit and bounded clearing is described without claiming guaranteed deletion.
- Analytics excludes authorities, portfolio/credential URIs, claims, nonces, presentation IDs, local record IDs, correlatable hashes, secrets, and raw attacker input.
- Privacy history records disclosed data, verifier authority, signed purpose, and time without adding undisclosed portfolio facts.
- Errors expose safe stable codes, render no attacker markup or secret, and use indistinguishable remote cryptographic failures where detailed causes would form an oracle.
- Destructive key, backup, record, and history operations identify exact scope and recovery consequences before confirmation.

Coverage: UX §16, §16.1–§16.3.
