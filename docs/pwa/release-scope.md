# PWA release scope

The first deployed release is a holder-first static PWA: onboarding, protected local authority setup, encrypted portfolio, acquisition, credential inspection, request validation, consent, presentation, privacy history, backup, and key/device management. A verifier workspace follows only after the holder release gate.

GitHub Pages contains no issuer or BVS signing service, provider OAuth secrets, administrative policy editor, total-loss recovery, selective disclosure, or proximity-security implementation. The application never claims conformance to the Working Drafts. The default project Pages deployment uses visibly synthetic data only.
