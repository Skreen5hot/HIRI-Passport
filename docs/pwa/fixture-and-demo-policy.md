# Fixture and demo policy

The public project Pages site uses only synthetic authorities, organizations, claims, endpoints, hashes, lifecycle evidence, requests, presentations, and policies. Fixtures are generated with project primitives and never copy placeholder signatures or hashes from specifications.

Valid, invalid, unknown, stale, replayed, tampered, offline, and unsupported cases remain separate. Production builds for a real-data origin disable demo loading and reset controls. A synthetic label remains visible throughout demo operation.
