# PWA information architecture

Holder routes are welcome, correlation notice, storage readiness, authority setup, backup, home, record detail, acquisition, self-assertion, request ingress, consent, signing, delivery receipt, privacy history, keys/devices, and settings. The verifier workspace is a later route group with import, progressive results, and export.

Every route defines loading, empty, valid, invalid, unknown, stale, offline, unsupported, expired/replayed, decline/cancel, and destructive-confirmation behavior as applicable. Routes use hash navigation so GitHub Pages never needs a server rewrite. Signed payloads do not enter URLs.
