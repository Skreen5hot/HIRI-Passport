# Build Week Submission: Known Limitations

## Synthetic data only

The submitted public application is a Synthetic Demo. It uses generated fixtures and non-authoritative example identifiers. Users and judges should not enter real personal information, credentials, private keys, authentication secrets, or production disclosure requests.

## Simulated workflows

Credential acquisition, consent, presentations, delivery, verification, key lifecycle, privacy history, offline behavior, and update behavior include synthetic or illustrative paths. The demo does not create a real holder authority or a protocol-authoritative signature.

## Evidence is deliberately separated

A displayed valid cryptographic result does not establish organizational identity, current status, provenance acceptance, or relying-party policy acceptance. The demo intentionally shows unknown, not-applicable, and not-evaluated results.

## Working Draft, no conformance claim

The Passport specifications are compatibility-corrected Working Drafts. They are not an adopted conformance target, certification program, or interoperability guarantee.

## Real Holder Preview is not submitted as production

The repository contains substantial fail-closed Real Holder Preview foundations. That runtime is not the working project represented in the Build Week video. Production configuration remains inert and the production resource catalog remains empty.

## Independent review pending

An exact resource candidate and complete independent-review kit exist, but the external human semantic review will occur after Build Week. Mechanical verification is not represented as independent approval. The candidate remains `candidateReady: false`, and no resource byte is activated as production normative material.

## Physical-device and operational evidence pending

Automated browser tests do not replace the planned physical-device matrix, backup-operator rehearsal, incident response evidence, protected release environment, rollback rehearsal, or final owner go/no-go decision.

## Hosting boundary

During judging, https://hiri-protocol.org/ serves the frozen Synthetic Demo. The post-challenge plan moves the demo to a separate origin before any Real Holder Preview deployment because `/demo/` on the same hostname would not isolate browser storage.

## No legal, regulatory, or real-world acceptance claim

The demo does not establish legal identity, professional standing, regulatory compliance, eligibility, authorization, or acceptance by any organization.
