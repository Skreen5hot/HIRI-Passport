# HIRI Passport

HIRI Passport is a user-controlled digital identity project for holding, presenting, and verifying signed claims with explicit consent and provenance.

## Specification status

Development is intentionally gated on the v2.0.0 specification revision.

- [`HIRI-Passport-v2_0_0-Compatibility-and-Normative-Decisions.md`](HIRI-Passport-v2_0_0-Compatibility-and-Normative-Decisions.md) is the accepted design-control baseline.
- [`HIRI-Digital-Passport-Extension-v2_0_0-DRAFT.md`](HIRI-Digital-Passport-Extension-v2_0_0-DRAFT.md) is the compatibility-corrected Working Draft. It is not yet a conformance target.
- [`HIRI-Passport-UX-Architecture-Spec-v2_0_0-DRAFT.md`](HIRI-Passport-UX-Architecture-Spec-v2_0_0-DRAFT.md) is the harmonized UX and application-architecture Working Draft.
- [`HIRI-Bootstrap-Verification-Profile-v3_0_0-DRAFT.md`](HIRI-Bootstrap-Verification-Profile-v3_0_0-DRAFT.md) is the harmonized, breaking BVS credential and evidence-profile Working Draft.

None of the Working Drafts is currently a conformance target. Development should use them together and must respect the open blockers recorded in the Core draft.

Historical design inputs are preserved unchanged and are not development contracts:

- [`HIRI-Digital-Passport-Extension-v1_9_0.md`](HIRI-Digital-Passport-Extension-v1_9_0.md)
- [`HIRI-Passport-UX-Architecture-Spec-v1_5_0.md`](HIRI-Passport-UX-Architecture-Spec-v1_5_0.md)
- [`HIRI-Bootstrap-Verification-Profile-v2_0_0.md`](HIRI-Bootstrap-Verification-Profile-v2_0_0.md)

The current compatibility baseline is pinned to HIRI Protocol v3.1.1 and HIRI Privacy & Confidentiality Extension v1.4.1 at upstream commit `009c145c9740188fc7a03b19c8ac2079bfe61cdb`.
